# Product Guidelines

## 1. 文章スタイルとコミュニケーション

- **トーン＆マナー**: 専門的かつ簡潔な表現を心がける。冗長な説明を省き、事実に基づいた客観的な記述を行う。
- **構造化**: 箇条書きやヘッダーを活用し、情報への到達性を高める。
- **UIテキスト**: 操作の結果や状態を、正確かつ短く伝える。

## 2. ユーザーインターフェース (UI) 設計

- **デザイン思想**: 機能性とミニマリズムを優先する。装飾よりも情報の密度と操作効率を重視した「管理ツール/IDE」スタイルを採用する。
- **一貫性**: Kobalte と Tailwind CSS をベースに、コンポーネントの挙動と視覚言語の一貫性を維持する。
- **パフォーマンス**: 大量データのリスト表示においても、レイアウトシフトを抑え、高速な応答性を維持する設計を行う。

## 3. コード品質と開発規則

- **堅牢性**: 厳格な型安全性を追求する。`as unknown as` 等のキャストを避け、Zodによるバリデーションと適切なエラーハンドリングを徹底する。
- **アーキテクチャの遵守**: クリーンアーキテクチャのレイヤー境界を尊重し、依存関係のルール（外側から内側へ）を厳守する。
- **自動化**: Biome によるリンティング・フォーマット、および Vitest によるテストを前提とし、CI/CDで品質を担保する。
