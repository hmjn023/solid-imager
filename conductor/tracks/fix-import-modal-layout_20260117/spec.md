# Specification: Fix Import Preview Modal Layout

## 1. Overview
インポートプレビューモーダル（`ImportPreviewModal`）において、コンテンツ量が多い場合に画面上部が切れて操作できなくなるレイアウト問題を修正する。

## 2. Functional Requirements

### 2.1 Layout Fix
- **Issue:** モーダルが `flex items-center` で垂直中央揃えされているため、高さが画面サイズに迫るとナビゲーションバーと重なったり、上部が見切れたりする。
- **Fix:**
    - モーダルの `z-index` をナビゲーションバー（`z-50`）より高い `z-[60]` 以上に設定する。
    - コンテンツ量が多くなっても、モーダルのヘッダー（タイトル、閉じるボタン）が常に画面上部から一定の距離を保って表示されるように配置を調整する。
    - `max-height` を適切に設定し（例: `90vh`）、内部のメディアリスト領域のみをスクロール可能にする。

## 3. Non-Functional Requirements
- **Visual:** 他の既存モーダル（`UploadMediaModal` 等）との統一感を持たせる。

## 4. Acceptance Criteria
- [ ] アイテム数が多いプレビュージョブを開いても、モーダルのヘッダーがナビゲーションバーの下、または前面に正しく表示され、操作（閉じる等）が可能であること。
- [ ] コンテンツが画面縦幅を超える場合、モーダル全体ではなくリスト領域がスクロールすること。
