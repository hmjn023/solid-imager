# DB操作関数の実装に関する調査

## 1. 目的

この調査の目的は、Drizzle ORMを使用して型安全で効率的なデータベースアクセスレイヤーを実装するための最適なアプローチを決定することです。

## 2. アプローチ

### 2.1. データアクセスレイヤーの構造

クリーンアーキテクチャの原則に基づき、データベース関連のロジックをインフラストラクチャ層に分離します。

- **ディレクトリ構造**: すべてのDB操作関数は、`src/infrastructure/db/queries/` ディレクトリ内に配置します。
- **ファイル分割**: `schema.ts` 内のテーブル定義に対応する形で、ドメイン（テーブル）ごとにクエリファイルを分割します。（例: `media.ts`, `tags.ts`）
- **再エクスポート**: `src/infrastructure/db/queries/index.ts` を作成し、すべてのクエリ関数を再エクスポートすることで、アプリケーションの他の部分からのインポートを簡素化します。

### 2.2. Drizzle ORMの活用

- **クエリビルダ**: Drizzle ORMが提供する型安全なクエリビルダを全面的に活用します。（例: `db.select().from(...)`, `db.insert(...)`）
- **リレーショナルクエリ**: `relations` を活用して、JOINを含む複雑なクエリを効率的に構築します。これにより、手書きのSQLを減らし、型安全性を維持します。
- **スキーマ**: `src/infrastructure/db/schema.ts` で定義されたテーブルスキーマと、そこから `InferSelectModel` / `InferInsertModel` を使って生成された型（`Media`, `NewMedia`など）を、関数の引数と戻り値の型として利用します。

## 3. 結論

上記のアプローチを採用することで、関心の分離が徹底され、メンテナンス性が高く、型安全なデータアクセスレイヤーを構築できると結論付けます。各関数は特定のテーブルに対する責務のみを持つため、コードの見通しが良くなります。
