refactor(db): Centralize database queries into a dedicated directory

This refactoring introduces a new `src/infrastructure/db/queries` directory to centralize all database access logic, improving code organization and adhering to cleaner architectural principles.

- **Move Existing Functions:** All database query functions previously located directly under `src/infrastructure/db/` (e.g., in `media.ts`, `categories.ts`) have been moved into the new `queries` directory, with each file corresponding to a specific domain/table.

- **Update Imports:** All import paths throughout the application have been updated to point to the new `queries` directory.

- **Add Integration Tests:** Created a suite of integration tests for the database functions to ensure their correctness and provide a safety net for this refactoring.

- **Cleanup:** Removed the old, now redundant, files from `src/infrastructure/db/`. Also removed an unused `tag.ts` file that was inconsistent with the project's tech stack.

- **Linting & Unit Tests:** Fixed all Biome linting errors, including file naming conventions, use of top-level regex, and removal of barrel files. Also refactored unit tests in `src/tests/db/` to correctly mock database interactions and resolve `TypeError: vi.mocked(...).mockImplementation is not a function` errors.

This change does not introduce any new functionality but significantly improves the maintainability and structure of the data access layer.
