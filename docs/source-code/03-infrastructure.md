# インフラストラクチャ層 (`src/infrastructure`)

このドキュメントでは、`src/infrastructure`ディレクトリ配下の各モジュールの役割と実装状況について解説します。

インフラストラクチャ層は、データベース、外部API、ファイルシステムといった、アプリケーションの外部にある技術的な詳細を実装する責務を持ちます。

## `api-clients`

-   **役割**: フロントエンドからバックエンドAPIを呼び出すためのクライアント関数群です。各ファイルが特定のエンドポイントグループに対応しています。
-   **実装状況**: **大半がモック実装**です。
    -   `categories.ts`, `characters.ts`, `ips.ts`, `tags.ts`: CRUD操作に対応する関数が定義されていますが、固定のダミーデータを返すモック実装です。
    -   `config.ts`: 設定の取得・更新・リセット関数がありますが、デフォルト設定を返すモック実装です。
    -   `events.ts`: SSE監視を開始する関数ですが、実際にはストリームを返さず、成功メッセージを返すだけです。
    -   `thumbnails.ts`: サムネイル生成開始・キャッシュクリアのAPIを呼び出す関数です。
    -   `directories.ts`, `media.ts`, `sources.ts`: 一部の関数は、他のレイヤー（`application/services`や`infrastructure/db`）を呼び出す、より具体的な実装がされています。

## `db`

-   **役割**: データベースとのやり取りをすべて担当します。Drizzle ORMを使用しています。
-   **実装状況**: **実装済み**
    -   `queries/`ディレクトリ配下の各ファイルには、テーブルごとのCRUD（作成、読み取り、更新、削除）操作を行うための具体的な関数が実装されています。これらはプレースホルダーではなく、Drizzle ORMを用いた完全なデータベースアクセスロジックです。
    -   `data-migration.ts`にも、データ移行のための具体的なロジックが実装されています。
    -   `index.ts`はデータベース接続を管理し、`errors.ts`はカスタムエラーを定義しており、これらも同様に実装済みです。
    -   `__mocks__/index.ts`はテスト用のモックであり、実際の機能とは関係ありません。

## `jobs`

-   **役割**: サムネイル生成などの時間のかかるバックグラウンド処理（ジョブ）を管理します。
-   **ファイル**:
    -   `job-queue.ts`: ジョブキューを管理するための汎用的なインターフェースですが、**未実装**のプレースホルダーです。
    -   `sse-manager.ts`: SSE（Server-Sent Events）を管理するためのインターフェースですが、**未実装**のプレースホルダーです。
    -   `thumbnail-jobs.ts`: サムネイル生成に特化した、インメモリのシンプルなジョブキューです。**実装済み**であり、ジョブの追加、処理、状態管理を行います。
    -   `thumbnails.ts`: `sharp`ライブラリを使用して、実際のサムネイル生成・削除処理を行います。**実装済み**です。

## `storage`

-   **役割**: `local`, `sftp`, `s3`といった異なる種類のメディアソースとのやり取りを抽象化する「ストレージドライバー」を実装します。
-   **ファイル**:
    -   `types.ts`: すべてのドライバーが実装すべき共通のインターフェース`MediaSourceDriver`を定義しています。
    -   `factory.ts`: メディアソースの種類に応じて、適切なドライバー（`LocalDriver`など）のインスタンスを返す`getDriver`関数を実装しています。
    -   `local.ts`: ローカルファイルシステムを操作するための`LocalDriver`を実装しています。**実装済み**です。
    -   `s3.ts`, `sftp.ts`: S3およびSFTPを操作するためのドライバーですが、**未実装**のプレースホルダーです。

## テスト

インフラストラクチャ層のコンポーネントは、インテグレーションテストによって検証されます。特に`db/queries`配下の関数は、実際のテスト用データベースに接続し、クエリが期待通りに動作することを確認します。

詳細は[テスト (`src/tests`)](./05-testing.md)のドキュメントを参照してください。
