# プレゼンテーション層 (`src/components`, `src/routes`)

このドキュメントでは、`src/components`および`src/routes`ディレクトリ配下の各コンポーネントやページの役割について解説します。

プレゼンテーション層は、ユーザーインターフェース（UI）とユーザー体験（UX）を担当します。

## `components`

### アプリケーション固有コンポーネント

-   **`Counter.tsx`**: クリックするとカウントが増えるシンプルなカウンターボタン。開発初期のテスト用コンポーネントです。
-   **`Nav.tsx`**: アプリケーションの主要ページ（Home, About, Sources）へのリンクを持つナビゲーションバーです。
-   **`SimpleModal.tsx`**: シンプルなモーダルダイアログを表示するためのテスト用コンポーネントです。
-   **`SourceCard.tsx`**: 単一のメディアソース情報を表示するためのカード型コンポーネントです。編集・削除ボタンも含まれます。
-   **`SourceDeleteModal.tsx`**: メディアソースを削除する際の確認モーダルです。
-   **`SourceFormModal.tsx`**: メディアソースの追加および編集を行うためのフォームを持つモーダルです。

### 汎用UIコンポーネント (`ui/`)

`shadcn/ui`の思想に基づいて`@kobalte/core`をラップした、再利用可能なUIコンポーネント群です。

-   `AlertDialog.tsx`: 警告や確認ダイアログを表示します。
-   `Button.tsx`: 様々なスタイル（variant）やサイズを持つ汎用ボタンです。
-   `Card.tsx`: コンテンツを囲むためのカードUIを提供します。
-   `Dialog.tsx`: 汎用的なモーダルダイアログです。
-   `Input.tsx`: 標準的なテキスト入力フィールドです。
-   `Label.tsx`: フォーム要素のラベルです。
-   `Select.tsx`: ドロップダウン選択メニューです。

## `routes`

SolidStartのファイルベースルーティングに従って、各ページを定義します。

-   **`index.tsx`**: ホームページ（`/`）です。
-   **`about.tsx`**: アプリケーションの概要ページ（`/about`）です。
-   **`sources.tsx`**: メディアソースを管理するためのメインページ（`/sources`）です。メディアソースの一覧表示、追加、編集、削除のUIロジックを実装しています。
-   **`[...404].tsx`**: 存在しないページにアクセスした際に表示される404 Not Foundページです。

## テスト

プレゼンテーション層のコンポーネントは、`@testing-library/solid`を用いたコンポーネントテストと、Playwrightを用いたE2Eテストによって検証されます。

-   **コンポーネントテスト**: 個々のUIコンポーネントが正しくレンダリングされ、インタラクションに反応するかを検証します。
-   **E2Eテスト**: ユーザーが実際に画面を操作するシナリオに沿って、ページ全体のフローが期待通りに動作するかを検証します。

詳細は[テスト (`src/tests`)](./05-testing.md)のドキュメントを参照してください。
