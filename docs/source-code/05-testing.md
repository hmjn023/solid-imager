# テスト (`src/tests`)

このドキュメントでは、`src/tests`ディレクトリに実装されているテストの種類、設定、および実装状況について解説します。

## テストの種類

このプロジェクトでは、品質を保証するために3種類のテストが導入されています。

1.  **ユニットテスト (`src/tests/unit`)**
    -   **目的**: 個々の関数やコンポーネント、サービスなどを単体で検証します。
    -   **特徴**: 依存関係（特にデータベースや外部API）を`vi.mock`を用いて完全にモック化し、テスト対象のロジックのみに集中します。

2.  **インテグレーションテスト (`src/tests/integration`)**
    -   **目的**: 複数のモジュールが連携して正しく動作するかを検証します。特に、データベースクエリが実際のDBに対して期待通りに動作することを確認します。
    -   **特徴**: `setup-integration.ts`で設定されたテスト用のデータベースに実際に接続してテストを実行します。テストの前後でデータのセットアップやクリーンアップを行います。

3.  **E2E (End-to-End) テスト (`src/tests/e2e`)**
    -   **目的**: ユーザーの視点から、アプリケーション全体のフローが正しく動作するかをブラウザを操作して検証します。
    -   **特徴**: Playwrightを使用し、実際のUI操作（クリック、フォーム入力など）をシミュレートして、画面の表示や遷移が期待通りであることを確認します。

## セットアップ

-   **`setup.ts`**: Vitestのグローバルセットアップファイル。テストの種類（ユニットかインテグレーションか）を判別し、データベースのモックを条件付きで適用するなどの環境設定を行います。
-   **`setup-unit.ts`**: ユニットテスト用の設定ファイル。データベース接続などをモックします。
-   **`setup-integration.ts`**: インテグレーションテスト用の設定ファイル。テスト用データベースへの接続情報を環境変数から読み込みます。

## 実装状況

-   **全体**: 多数のテストファイルが作成されていますが、その多くは初期段階のプレースホルダーです。
-   **ユニットテスト**: `media` APIクライアントなど、一部のロジックに対しては具体的なテストが実装されています。
-   **インテグレーションテスト**: `media`テーブルのクエリなど、データベースの主要な操作に関してはテストが実装されています。
-   **E2Eテスト**: メディアソース管理画面の基本的なCRUD操作（作成、編集、削除）に関するテストが実装されています。

**結論**: プロジェクトの主要な機能領域において、各テストレベルでの実装が着手されていますが、網羅的なテストスイートとしてはまだ発展途上です。
