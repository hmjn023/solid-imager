# ソースコード概要 (`src`)

このドキュメントでは、`src`ディレクトリ以下のソースコードの構造と設計思想について概説します。

## ディレクトリ構造

`src`ディレクトリは、クリーンアーキテクチャの考え方に基づき、以下の主要なレイヤーに分割されています。

-   **`application`**: アプリケーション固有のビジネスロジックを配置します。UI（プレゼンテーション層）からの要求を受け取り、ドメイン層のエンティティやサービスを組み合わせてユースケースを実現します。
-   **`domain`**: アプリケーションの核となるビジネスルールとエンティティを定義します。このレイヤーは他のどのレイヤーにも依存しません。
-   **`infrastructure`**: データベース、外部API、ファイルシステムなど、アプリケーションの外部にある技術的な詳細を実装します。ドメイン層で定義されたインターフェースを実装する形をとります。
-   **`components`**: UIを構成する再利用可能なSolidJSコンポーネントを配置します。
-   **`routes`**: SolidStartのファイルベースルーティングに対応するファイル群です。UIのページやAPIエンドポイントを定義します。

## 設計思想

-   **クリーンアーキテクチャ**: 関心事の分離を徹底し、ドメインロジックをフレームワークやUI、データベースから独立させることを目指します。これにより、テスト容易性、保守性、将来の技術変更への対応力を高めます。
-   **ドメイン駆動設計 (DDD)**: `domain`レイヤーを中心に、ビジネスの関心事をモデル化します。
-   **型安全性**: TypeScriptとDrizzle ORMを全面的に採用し、コンパイル時および実行時の型安全性を確保します。Zodによるバリデーションも活用します。

## 実装状況に関する注意

現在、多くのファイル、特に`application/services`配下のサービスクラスは、プレースホルダー（`throw new Error("Not implemented")`）として実装されています。これは、まず全体のアーキテクチャとインターフェースを定義し、その後で具体的な実装を進める開発アプローチに基づいています。

各機能の具体的な実装状況については、このドキュメントシリーズの各ファイルを参照してください。
