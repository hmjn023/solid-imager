
# 概要

ComfyUIで生成した画像・動画などのメディアを管理するためのバックエンドAPIシステム。

### 主要機能
- メディアソース管理（ローカル/SFTP/S3）
- メディア配信・サムネイル生成
- メタデータ抽出（prompt/workflow）
- リアルタイム更新通知（SSE）
- メディアアップロード・編集・一括ダウンロード
- 検索・ソート・タグ管理
- AIタグ付け・キャラクター抽出
- 設定管理

### 用語定義
- **メディア**: 画像、動画、音声などのファイル
- **メディアソース**: ファイルが保存されている場所、及びその管理形態
- **サブディレクトリ**: 特定のメディアソース内にあるディレクトリ、及びその内部の再帰的なディレクトリ
- **メディア一覧**: 特定のメディアソースに保存されているメディアやそのサブディレクトリをグリッド状に表示する画面
- **メディア情報**: 名称、メディアのサイズ、ファイルサイズ、作成日時、更新日時などの基本的な情報に加え、評価、閲覧回数、技術的な詳細（EXIFデータ、ハッシュなど）を含む、メディアに関する様々な付加情報。
- **生成情報**: メディアが生成された際にPNGのtEXt領域に保存される情報（プロンプト、ワークフロー）に加え、使用されたモデル名、シード値、CFGスケール、ステップ数などを含む、AI生成に関する詳細情報。
- **カテゴリ**: メディアを分類するための分類項目。`categories` テーブルで管理され、階層構造を持つことができます。
- **タグ**: メディアに複数関連付けられるキーワード情報。`tags` テーブルと `media_tags` テーブルで管理され、メディアの検索や分類に利用されます。
- **IP (知的財産)**: メディアが関連する作品やシリーズなどの知的財産。`ips` テーブルで管理され、キャラクターやメディアに紐づくことがあります。
- **キャラクター**: メディアに登場する人物や架空の存在。`characters` テーブルで管理され、特定のIPに紐づく場合があります。
- **プロジェクト**: メディアを特定の作業や目的のためにまとめる単位。`projects` テーブルで管理されます。
- **コレクション**: ユーザーが作成するメディアのまとまり。`collections` テーブルで管理され、複数のメディアを含むことができます。
- **ユーザー**: システムを利用する個人。`users` テーブルで管理され、コレクションなどの所有者となります。
- **ジョブ (Job)**: サムネイル生成、メタデータ抽出、一括操作など、時間のかかる非同期処理の単位。`jobs` テーブルで管理されます。
- **プリセット (Preset)**: よく使う検索条件を保存したもの。`presets` テーブルで管理され、フィルター操作を効率化します。

### 技術単語
- **リアルタイム更新通知（SSE）**: Server-Sent Events (SSE) を利用して、ファイルシステムの変更（追加、削除、変更）やサムネイル生成の進捗状況などをクライアントにリアルタイムで通知する機能。
- **パスのワイルドカード (`*`)**: APIエンドポイントのパスにおいて、特定のディレクトリ以下の「完全な相対パス」を表すために使用される記号。SolidStartのルーティングでは `[...param]` と記述されます。
